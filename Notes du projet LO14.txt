Il faut créer un serveur d'archive. Utiliser le code de base du serveur du prof
Réussir à communiquer avec ce serveur en éditant une nouvelle commande lancée par le client
Editer les fonctions dans le code du prof pour permettre au serv de reconnaitre les commandes lancées par le client
On interroge le serveur d'archive avec une commande crée

La commande doit avoir 4 modes et respecter une synthaxe ( Elle doit pouvoir lancer 4 actions différentes ) :

  - Mode list -> Afficher la liste des archives (ls) présente sur le serveur (On peut créer les archives à la main)
  - Mode create -> Créer une archive portant comme nom l'argument écrit par le client lors de l'appel de la fonction (Le serv doit reconnaitre les arguments donc) de l'arborescence du répertoire courant du client
  - Mode browse -> Explorer le serveur d'archive, naviguer dedans (cd) en explorant une archive renseignée en argument ( son nom)
  - Mode Extract -> Extraire le contenue de l'archive dont le nom est renseignée en argument vers le répertoire courant de la machine cliente.

Archive = Fichier qui représente l'arboresence d'un répertoire (Ce qu'il contient) et le contenu de tous les fichiers de cette arbosrences (Cat)

Archive = Header + Body

Header -> arboresence des fichiers
Body -> Contenu des différents fichiers

Header c'est un ensemble de répertoires juxtaposés les uns en dessous des autres 
Body contient les contenus de tous les fichiers non vides de l'arboresences décrite dans le header.

La commande browse doit pouvoir faire du pwd, du ls, du cd, du cat, du rm, du touch et du mkdir
